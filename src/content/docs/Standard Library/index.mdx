The standard library is currently in development, so frequent changes will occur. Note that all std::core modules and
sub modules are implicitly imported.

:::caution
This page is missing many new parts of the standard library, see the [Standard Library Reference](/standard-library/stdlib_refcard/) for a more up-to-date overview
:::

## `std::core::builtin`

All functions and macros in this library can be used without path qualifiers.

### `void panic(String message, String file, String function, uint line)`
Default function called when the asserts fails.

### `void @swap(#a, #b)`
Swap values in `a` and `b`.

```c
int a = 3;
int b = 5;
@swap(a, b);
io::printfn("%d", a); // Prints 5
```

### `anycast(any v, $Type)`

Optionally cast the value `v` to type `$Type*` on failure returns `TYPE_MISMATCH`.

```c
int b;
any a = &b;
float*? c = anycast(a, float); // Will return TYPE_MISMATCH
int*? d = anycast(a, int);     // Works!
```

### `void unreachable($string = "Unreachable statement reached.")`

Mark a code path as unreachable.

```c
switch (x)
{
    case 0:
        foo();
    case 1:
        bar();
    default:
        // Should never happen.
        unreachable("x should have been 0 or 1");
}
```

On safe mode this will throw a runtime panic when reached. For release mode the
compiler will assume this case never happens.

### `bitcast(value, $Type)`
Do a bitcast of a value to `$Type`, requires that the types are of the same memory size.
```c
float x = 1.0;
int y = bitcast(x, int); // y = 0x3f800000
```

### `enum_by_name($Type, enum_name)`
Optionally returns the enum value with the given name. `$Type` must be an enum. Returns `SearchResult.MISSING`
on failure.
```c
enum Foo { ABC, CDE, EFG }

fn void? test()
{
    Foo f = enum_by_name(Foo, "CDE")!;
    // same as Foo f = Foo.CDE;
}
```

### `void @scope(#variable; @body)`

Scopes a variable:

```
int a = 3;

@scope(a)
{
    a = 4;
    a++;
};

// Prints a = 3
io::printfn("a = %d", a, b);
```

### `less`, `greater`, `less_eq`, `greater_eq`, `equals`
All macros take two values and compare them. Any type implementing `Type.less`
or `Type.compare_to` may be compared (or if the type implements `<`). Types
implementing `Type.equals` may use `equals` even if neither `less` nor `compare_to`
are implemented.

## `std::core::env`

### Constants
- `OS_TYPE` the OS type compiled for.
- `COMPILER_OPT_LEVEL` the optimization level used.
- `I128_SUPPORT` true if int128 support is available.
- `COMPILER_SAFE_MODE` true if compiled with safety checks.

## `std::core::mem`

### `malloc`, `malloc_aligned`

Allocate the given number of bytes. both `malloc` and `malloc_aligned` will panic on out of memory.
`malloc_aligned` adds an alignment, which must be a power of 2. Any pointer
allocated using `malloc_aligned` must be freed using `free_aligned` rather
the normal `free` or memory corruption may result.

```c
char* data = malloc(8);
int[<16>]* data2 = malloc_aligned(16 * int.sizeof, 128);
```

### `new($Type, #init = ...), new_aligned($Type, #init = ...)`

This allocates a single element of $Type, returning the pointer. An optional initializer may be added, which
immediately initializes the value to that of the initializer.

If no initializer is provided, it is zero initialized. `new_aligned` works the same but for overaligned types, such allocations
must be freed using `free_aligned`

```c3
int* a = mem::new(int);
Foo* foo = mem::new(Foo, { 1, 2 });
```

### `alloc($Type)`, `alloc_aligned($Type)`

Allocates a single element of $Type, same as `new`, but without initializing the data.

### `new_array($Type, usz elements)`, `new_array_aligned($Type, usz elements)`

Allocates a slice of `elements` number of elements, returning
a slice of the given length. Elements are zero initialized. `new_array_aligned` is used for
types that exceed standard alignment.

```c3
int[] ints = mem::new_array(int, 100); // Allocated int[100] on the heap, zero initialized.
```

### `alloc_array($Type, usz elements)`, `alloc_array_aligned($Type, usz elements)`

Same as `new_array` and `new_array_aligned` but without initialization.

### `calloc`, `calloc_aligned`

Identical to the `malloc` variants, except the data is guaranteed to be zeroed out.

### `realloc`, `realloc_aligned`

Resizes memory allocated using `malloc` or `calloc`. Any extra data is
guaranteed to be zeroed out. `realloc_aligned` can only be used with
pointers created using `calloc_aligned` or `alloc_aligned`.

### `free`, `free_aligned`

Frees memory allocated using `malloc` or `calloc`. Any memory allocated using "_aligned" variants
must be freed using `free_aligned`.

### `@scoped(Allocator allocator; @body())`

Swaps the current memory allocator for the duration of the call.

```c
DynamicArenaAllocator dynamic_arena;
dynamic_arena.init(1024);
mem::@scoped(&dynamic_arena)
{
    // This allocation uses the dynamic arena
    Foo* f = malloc(Foo);
};
// Release any dynamic arena memory.
dynamic_arena.destroy();

```

### `void* tmalloc(usz size, usz alignment = 0)`

Allocates memory using the temporary allocator. Panic on failure. It has type
variants similar to `malloc`, so `tnew(Type)` would create a `Type*` using
the temporary allocator.

### `void* tcalloc(usz size, usz alignment = 0)`

Same as `tmalloc` but clears the memory.

### `void* trealloc(void* ptr, usz size, usz alignment = 0)`

`realloc` but on memory received using `tcalloc` or `tmalloc`.

### `tnew`, `temp_alloc`, `temp_array`, `talloc_array`

Same as the `new`, `alloc`, `new_array` and `alloc_array` respectively.

### `void @pool(;@body)`

Opens a temporary memory scope.

```c3
@pool()
{
    // This allocation uses the dynamic arena and is freed automatically at the end of the block
    Foo* f = tmalloc(Foo);
};
```


### `@volatile_load(#x)`

Returns the value in `#x` using a volatile load.

```c3
// Both loads will always happen:
int y = @volatile_load(my_global);
y = @volatile_load(my_global);
```


### `@volatile_store(#x, y)`

Store the value `y` in `#x` using a volatile store.

```c3
// Both stores will always happen:
@volatile_store(y, 1);
@volatile_store(y, 1);
```
### `usz aligned_offset(usz offset, usz alignment)`

Returns an aligned size based on the current offset. The alignment
must be a power of two. E.g. `mem::aligned_offset(17, 8)` would return `24`

### `usz aligned_pointer(void* ptr, usz alignment)`

Returns a pointer aligned to the given alignment, using `aligned_offset`.

### `bool ptr_is_aligned(void* ptr, usz alignment)`

Return true if the pointer is aligned, false otherwise.

### `void copy(void* dst, void* src, usz len, usz $dst_align = 0, usz $src_align = 0, bool $is_volatile = false)`

Copies bytes from one pointer to another. It may optionally be set as volatile,
in which case the copy may not be optimized away. Furthermore the source
and destination alignment may be used.

```c3
Foo* f = tmalloc(data_size);
mem::copy(f, slice.ptr, size);
```

### `void set(void* dst, char val, usz len, usz $dst_align = 0, bool $is_volatile = false)`

Sets bytes to a value. This operation may be aligned and/or volatile. See the `copy` method.

### `void clear(void* dst, usz len, usz $dst_align = 0, bool $is_volatile = false)`

Sets bytes to zero. This operation may be aligned and/or volatile. See the `copy` method.

### `@clone(&value)`

Makes a shallow copy of a value using the regular allocator.

```c
Foo f = ...

return @clone(f);
```

### `@tclone(&value)`

Same as `@clone` but uses the temporary allocator.

## `std::core::types`

### `bool is_comparable_value($Type)`

Return true if the type can be used with comparison operators.

### `bool is_equatable_value(value)`

Return `true` if the value can be compared using the `equals` macro.

### `bool is_equatable_value(value)`

Return `true` if the value can be compared using the comparison macros.

### `kind_is_int(TypeKind kind)`
### `any_to_int(any v, $Type)`

Returns an optional value of `$Type` if the any value losslessly
may be converted into the given type. Returns either `VALUE_OUT_OF_RANGE` or `VALUE_OUT_OF_UNSIGNED_RANGE` otherwise.

```c
any v = &&128;
short y = any_to_int(v, short)!!; // Works
ichar z = any_to_int(v, ichar)!!; // Panics VALUE_OUT_OF_RANGE
```

## `std::core::str::conv`

### `usz? char32_to_utf8(Char32 c, char[] output)`
Convert a UTF32 codepoint to an UTF8 buffer. It returns the number of bytes used, or
`string::CONVERSION_FAILED` if the buffer is too small.

### `void char32_to_utf16_unsafe(Char32 c, Char16** output)`
Convert a UTF32 codepoint to an UTF16 buffer without bounds checking,
moving the output pointer 1 or 2 steps.

## `std::io`

### `String? readline(Allocator allocator, stream = io::stdin())`
Read a `String?` from a file stream, which is standard input (stdin) by default, reads to the next newline character `\n` or to the end of stream.
`readline` returns an [Optional](/language-common/optionals-essential/#what-is-an-optional) string.

```c3
import std::io;

fn void? hello_name()
{
    String? name = io::readline(mem); // Warning: leaking memory
    if (catch excuse = name)
    {
        return excuse?;
    }

    io::printfn("Name was: %s.", name);
}
```

:::note
`\r` will be filtered from the String.
:::

### `String? treadline(stream = io::stdin())`
Read a `String?` from a file stream which is standard input (stdin) by default, Reads to the next newline character `\n` or to the end of stream. `Treadline` returns an [Optional](/language-common/optionals-essential/#what-is-an-optional) string. The temporary allocator is used by `treadline`, in contrast the `readline` can be used with any allocator.

```c
import std::io;

fn void? hello_name()
{
    String? name = io::treadline();
    if (catch excuse = name) {
        return excuse?;
    }

    io::printfn("Hello %s! Hope you have a great day", name);
}
```

:::note
`\r` will be filtered from the String.
:::

### `void print(x)`, `void printn(x = "")`

Print a value to stdout works for the majority of types, including structs, which can be helpful for debugging.
The `printn` variant appends a newline.

```c3
import std::io;

enum Heat
{
    WARM,
    WARMER,
    REALLY_WARM,
}

fn void main()
{
    int[<2>] vec = { 4, 2 };
    Heat weather = WARM;
    int[5] fib = { 0, 1, 1, 2, 3 };
    String dialogue = "secret";

    io::print("Hello");   // Hello
    io::print(20);        // 20
    io::print(2.2);       // 2.200000
    io::print(vec);       // [<4, 2>]
    io::print(weather);   // WARM
    io::print(fib);       // [0, 1, 1, 2, 3]
    io::print(dialogue);  // secret
}
```

### `void eprint(x)`, `void eprintn(x)`
Print any value to stderr.
The `eprintn` variant appends a newline.

See `print` for usage.

### `usz? printf(String format, args...) @maydiscard`
Regular printf functionality: `%s`, `%x`, `%d`, `%f` `%p`, `%h`, and `%c` are supported.
Will also print enums and vectors. Prints to stdout.

```c3
import std::io;

{
    WARM,
    WARMER,
    REALLY_WARM,

{
    int[<2>] vec = { 4, 2 };

    String dialogue = "Hello";

    io::printfn("%s", dialogue);  // Hello

    io::printfn("%f", 2.2);       // 2.200000
    io::printfn("%s", vec);       // [<4, 2>]
    io::printfn("%s", weather);   // REALLY_WARM

```

Also available as `printfn` which appends a newline.

## `std::collections::orderedmap{Key, Value}`

`OrderedMap` is a map for cases where key order matters. Unlike a hash map, iteration is sorted by key, which makes it useful for indexes, interval tables, ranked values, and any workload where you want to ask for "the next key", "the previous key", or "everything between A and B".

Internally it is implemented as a skip list. In everyday use, the important part is the behavior:

- lookup by key
- iteration in key order
- access to neighboring keys
- range iteration without sorting a separate list first

By default `OrderedMap` uses the built-in ordering for `Key`, so the key type must support ordered comparison. If you want a different order, for example descending or case-insensitive, initialize it with a custom comparator.

Typical lookup, insert, and remove operations are expected `O(log n)`. Iteration is `O(n)`, range iteration is `O(log n + k)` for `k` returned elements, and space usage is `O(n)`.

### When To Use It

Use `OrderedMap` when you need both mapping and ordering.

- Use `std::collections::map` when fast key lookup matters but key order does not.
- Use `OrderedMap` when iteration order should follow the keys.
- Use `OrderedMap` when you need nearest-match queries such as floor, ceiling, lower, or higher.
- Use `OrderedMap` when you want to iterate only part of the key space.

### Basic Usage

```c3
import std::collections::orderedmap;
import std::io;

alias IntToStringMap = OrderedMap{int, String};

fn void main()
{
    IntToStringMap map;
    map.init(mem);
    defer map.free();

    map[10] = "ten";
    map[3] = "three";
    map[7] = "seven";

    foreach (entry : map.iter())
    {
        io::printfn("%s -> %s", entry.key, entry.value);
    }
}
```

The output is ordered by key:

```text
3 -> three
7 -> seven
10 -> ten
```

Even though the values were inserted in the order `10`, `3`, `7`, iteration follows the key order.

### Common Operations

```c3
if (try name = map[7])
{
    io::printfn("found: %s", name);
}

bool updated = map.set(7, "SEVEN");
bool exists = map.has_key(3);
bool removed = map.remove(10);
```

The main operations are:

- `map[key]` returns `NOT_FOUND` if the key is absent.
- `map[key] = value` inserts or updates.
- `set` returns `true` when an existing key was updated, and `false` when a new entry was created.
- `get_ref` is useful when you want to mutate the stored value in place.
- `@get_or_set` is useful for lazy initialization.

Example:

```c3
alias CounterMap = OrderedMap{String, int};

CounterMap map;
map.init(mem);
defer map.free();

int* count = map.get_ref("warnings") ?? {
    map["warnings"] = 0;
    return map.get_ref("warnings")!!;
};
(*count)++;
```

### Nearest-Key Queries

One of the main reasons to use `OrderedMap` over a hash map is the support for neighbor queries:

- `floor_key(k)` gives the greatest key less than or equal to `k`
- `ceiling_key(k)` gives the least key greater than or equal to `k`
- `lower_key(k)` gives the greatest key strictly less than `k`
- `higher_key(k)` gives the least key strictly greater than `k`

This is useful for table lookup and range navigation:

```c3
alias PriceTable = OrderedMap{int, String};

PriceTable prices;
prices.init(mem);
defer prices.free();

prices[10] = "basic";
prices[25] = "pro";
prices[100] = "enterprise";

if (try tier_key = prices.floor_key(30))
{
    io::printfn("matched tier: %s", prices[tier_key]!!);
}
```

### Range Iteration

If you only want part of the map, you do not need to iterate everything.

- `iter()` walks the whole map in key order
- `iter_from(k)` starts at the first key greater than or equal to `k`
- `iter_range(a, b)` walks the half-open range `[a, b)`

```c3
foreach (entry : map.iter_range(10, 20))
{
    io::printfn("%s = %s", entry.key, entry.value);
}
```

This makes `OrderedMap` a practical choice for paged output, time windows, or prefix-like numeric ranges.

### Custom Ordering

To change iteration order, pass a comparator to `init_with_cmp`. The comparator should return `true` when the first argument should come before the second.

```c3
fn bool descending(int a, int b)
{
    return a > b;
}

OrderedMap{int, String} map;
map.init_with_cmp(mem, descending);
defer map.free();

map[1] = "one";
map[3] = "three";
map[2] = "two";

foreach (entry : map.iter())
{
    io::printfn("%s", entry.key);
}
```

This prints `3`, `2`, `1`.

### Lifecycle

Initialize the map with `init`, `tinit`, `init_with_cmp`, or `tinit_with_cmp`.

- `clear()` removes all entries but keeps the map allocated.
- `free()` releases all memory owned by the map.
- `clone()` and `tclone()` create a copy of the map.
- `merge()` inserts all entries from another ordered map and overwrites on key collision.
- `first_key()`, `last_key()`, `pop_first()`, and `pop_last()` are useful when treating the map as a sorted work queue.

### API Summary

- `init` / `tinit`: create a map with default ordering
- `init_with_cmp` / `tinit_with_cmp`: create a map with custom ordering
- `get`, `get_ref`, `set`, `has_key`, `remove`: basic map operations
- `first_key`, `last_key`, `floor_key`, `ceiling_key`, `lower_key`, `higher_key`: ordered navigation
- `iter`, `iter_from`, `iter_range`: ordered traversal
- `len`, `is_empty`, `clear`, `free`: container management
- `clone`, `tclone`, `merge`: copying and combining maps

### API Reference

#### `OrderedMap.init(&self, Allocator allocator)`, `tinit(...)`

Initialize the map using the provided allocator or the temporary allocator.

#### `OrderedMap.init_with_cmp(&self, Allocator allocator, CmpFn cmp_fn)`, `tinit_with_cmp(...)`

Initialize the map with a custom comparator. The comparator should return `true` when the first key should come before the second key.

#### `bool OrderedMap.set(&self, Key key, Value value)`, `map[key] = value`

Insert or update an entry. Returns `true` if an existing key was updated, `false` if a new entry was inserted.

#### `Value? OrderedMap.get(&self, Key key)`, `map[key]`

Get the value for a key, or `NOT_FOUND` if the key does not exist.

#### `Value*? OrderedMap.get_ref(&self, Key key)`, `map&[key]`

Get a pointer to the stored value for in-place updates.

#### `Value OrderedMap.@get_or_set(&self, Key key, #default_value)`

Return the current value for `key`, or insert and return `#default_value` if the key is missing.

#### `bool OrderedMap.has_key(&self, Key key)`

Return `true` if the key exists.

#### `bool OrderedMap.remove(&self, Key key)`

Remove a key and return whether it was present.

#### `Key? OrderedMap.first_key(&self)`, `last_key(&self)`

Return the first or last key in sorted order.

#### `Entry? OrderedMap.pop_first(&self)`, `pop_last(&self)`

Remove and return the first or last entry in sorted order.

#### `Key? OrderedMap.floor_key(&self, Key key)`

Return the greatest key less than or equal to `key`.

#### `Key? OrderedMap.ceiling_key(&self, Key key)`

Return the least key greater than or equal to `key`.

#### `Key? OrderedMap.lower_key(&self, Key key)`

Return the greatest key strictly less than `key`.

#### `Key? OrderedMap.higher_key(&self, Key key)`

Return the least key strictly greater than `key`.

#### `sz OrderedMap.len(&self)`, `bool OrderedMap.is_empty(&self)`

Return the number of entries or whether the map is empty.

#### `void OrderedMap.clear(&self)`, `free(&self)`

Remove all entries, or release all memory owned by the map.

#### `OrderedMap OrderedMap.clone(&self, Allocator allocator)`, `tclone(&self)`

Clone the map into a new map. The clone keeps the same comparator.

#### `void OrderedMap.merge(&self, OrderedMap* other)`

Insert all entries from `other`, overwriting existing values on key collisions.

#### `OrderedMapIterator OrderedMap.iter(&self)`

Iterate over all entries in sorted order.

```c3
foreach (entry : map.iter())
{
    io::printfn("%s = %s", entry.key, entry.value);
}
```

#### `OrderedMapRangeIterator OrderedMap.iter_from(&self, Key from_key)`

Iterate from the first key greater than or equal to `from_key`.

#### `OrderedMapRangeIterator OrderedMap.iter_range(&self, Key from_key, Key to_key)`

Iterate over keys in the half-open range `[from_key, to_key)`.

```c3
foreach (entry : map.iter_range(10, 20))
{
    io::printfn("%s = %s", entry.key, entry.value);
}
```
